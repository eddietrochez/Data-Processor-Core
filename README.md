# Data-Processor-Core 🚀

A hybrid backend solution designed to bridge the gap between robust enterprise systems and high-performance modern architectures.

## 🎯 Project Purpose
This repository serves as a professional showcase of a **Modernization & Performance Architecture**:
1.  **Core Processing (C++):** Optimized logic for high-speed computational tasks and data processing.
2.  **Integration Layer (Python):** Orchestrating data from enterprise sources (like IBM i / AS/400) into standardized formats for modern applications.

## 🛠️ Tech Stack
*   **C++ (Modern Standards):** Used for the core processing engine.
*   **Python 3.x:** Used for integration, API-ready data structures, and reporting.
*   **GitHub Actions Ready:** Designed for CI/CD pipeline integration.

## 🏛️ Enterprise Systems (Legacy & Modernization)
*   **IBM i (AS/400):** Basic to Intermediate navigation in "Green Screen" environments. 
*   **CL (Control Language):** Executing and managing system commands for backend operations.
*   **Query/400:** Currently developing skills in data extraction and custom query creation to bridge legacy data with modern analytics.

## 📂 Project Structure
*   `processor.cpp`: High-performance core that handles heavy computational logic.
*   `main.py`: Integration script providing a Live Data Entry system and report orchestration.
*   `output_report.json`: Standardized output for modern web services and APIs.
*   `output_report.csv`: Enterprise-ready format designed for Excel, data analysis, and legacy system integration (AS/400).

## ⚙️ How it Works
The C++ core processes raw data streams, which are then captured by the Python layer to be transformed into standardized formats (JSON), simulating a real-world enterprise backend environment.

## 🖥️ How to Run

### 1. Execute the C++ Core
Compile and run the performance engine:
```bash
g++ processor.cpp -o processor && ./processor
```

### 2. Run the Python Integration
Execute the reporting script and enter the metrics when prompted:
```bash
python3 main.py
```

### 3. Check Results
View the generated reports in your folder:
* `output_report.json`
* `output_report.csv`
